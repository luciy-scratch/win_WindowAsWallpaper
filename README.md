# Win_WindowAsWallpaper (WAW)

指定した GUI アプリケーションを Windows デスクトップの壁紙レイヤー（WorkerW）に埋め込み、背景として常駐させるツールです。

## 主な機能

- **壁紙化**: ウィンドウの枠線やタイトルバーを消去し、デスクトップアイコンの背後に配置します。
- **グリッド配置**: 各モニタを縦 2 × 横 4 のグリッドで分割管理し、座標指定で正確にレイアウト可能です。
- **プロセス管理**: Job Object を使用し、本ツール終了時に配置したアプリも一括でクリーンアップします。
- **システムトレイ常駐**: 実行中はタスクバーの通知領域に格納され、コンソールを隠して動作します。

## 動作環境

- **OS**: Windows 11 (推奨) / Windows 10
- **権限**: 管理者権限 (タスクマネージャーなどの操作や、一部のアプリ起動に必要です)
- **Python**: 3.13.13 (Embeddable Package 64bit)

## 導入方法

### 1. Python と uv.exe の準備

1. Python 3.13.13 Embeddable Package (64bit) をダウンロードし、Zipファイルを解凍したフォルダをプロジェクト直下に配置します。<br>
   https://www.python.org/downloads/latest/python3.13/
2. uvのリポジトリから最新のリリースをダウンロードし、`uv.exe`をプロジェクト直下に配置します。<br>
   https://github.com/astral-sh/uv/releases/latest/

### 2. 環境の初期化とライブラリのインストール

ターミナルでプロジェクトフォルダに移動し、以下のコマンドを実行して環境を構築します。

```bash
# uv環境の初期化
./uv.exe init --bare --python [プロジェクト直下のpythonのフォルダパス]

# venv環境の初期化
./uv.exe venv --python [プロジェクト直下のpythonのフォルダパス]

# 依存ライブラリのインストール
./uv.exe add --link-mode=copy pywin32 pystray Pillow rich
```

### 2. アニメーション設定の確認

ウィンドウが正しく壁紙レイヤーに移動するために、Windows の「システムの詳細設定」→「パフォーマンス」で以下の設定が有効であることを確認してください。

- **「ウィンドウを最大化や最小化するときにアニメーションで表示する」をチェック**

> [!warning]
>
> 現状、"この設定を有効にしたら動いた"程度の確認のみです。
> もしかしたら、同じく「パフォーマンス」の設定項目から別の項目を有効にしてある必要があるかもしれません。
> 更なる要件が発覚すればここに追記します。

## 実行方法

1. 同梱された `WAW_runner.bat` を**管理者権限**で実行します。
2. 起動後、`settings.json` の内容に従ってアプリが順次起動し、配置されます。
3. 全ての配置が完了すると、コンソールが隠れ、システムトレイ（通知領域）にアイコンが表示されます。
4. 終了する場合は、トレイアイコンを右クリックして **[Exit]** を選択してください。

## 設定の変更 (settings.json)

初回起動時にサンプルが生成されます。以下のパラメータを編集してカスタマイズしてください。


| キー      | 説明                                                                           |
| :---------- | :------------------------------------------------------------------------------- |
| `path`    | 実行ファイルの絶対パス、またはシステムパスが通っている名前。                   |
| `args`    | 起動時に渡す引数。                                                             |
| `title`   | ウィンドウ特定のためのタイトル名（部分一致）。リダイレクトされるアプリに有効。 |
| `monitor` | 表示するモニタ番号 (0 から開始)。                                              |
| `x`, `y`  | 配置するグリッドの左上座標。x は 0～3, y は 0～1。                             |
| `w`, `h`  | ウィンドウのサイズ（グリッド単位）。w は 1～4, h は 1～2。                     |
| `wait_ms` | 起動からウィンドウ検出までの最大待機時間（ミリ秒）。                           |
| `taskbar` | `1` のときタスクバーを除いた領域で計算、`0` のときモニタ全体で計算。           |

### グリッドシステムの例

モニタ 1 枚を以下の 8 エリアに分割して管理します。

```
[x0,y0][x1,y0][x2,y0][x3,y0]
[x0,y1][x1,y1][x2,y1][x3,y1]
```

## 使用上の注意点

### 1. UWP アプリ (ストアアプリ) 非対応

Windows 11 の「メモ帳」「タスクマネージャー」「電卓」などの UWP/WinUI3 ベースのアプリは、描画の仕組みが特殊なため、壁紙レイヤーへの埋め込み後に中身が描画されない場合があります。
**旧来の Win32 アプリケーション（`conhost.exe` 経由のコマンドプロンプト等）での使用を推奨します。**

### 2. 管理者権限での実行

壁紙として配置したいアプリケーションが管理者権限で動作している場合、本ツールも管理者権限で実行しないと、ウィンドウのハンドル操作やスタイルの変更が拒否されます。

### 3. ゴーストウィンドウ

一部のアプリケーションは終了時にプロセスが残ることがあります。本ツールは終了時に極力起動したプロセスをキルするように設計されていますが、強制終了が困難なアプリがある場合は手動でタスクを終了してください。

## 開発者向け

このツールは `WorkerW` という Windows の特殊なウィンドウ階層を利用しています。Wallpaper Engine 等の動的壁紙ソフトが既に起動している場合は、そのレイヤーを自動的に検出して相乗りするように動作します。
